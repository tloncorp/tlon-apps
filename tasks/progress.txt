## Codebase Patterns
- Inline types are defined in `packages/shared/src/urbit/content.ts` with type guards like `isBold`, `isItalics`, etc.
- Story/Verse types are defined in `packages/shared/src/urbit/channel.ts`
- Use Vitest for tests in `packages/shared` - import from `vitest`
- Test files use `.test.ts` suffix in same directory as source files
- Block types include: Image, Header, Rule, Code, ListingBlock (contains Listing = List | ListItem)
- List structure: `{ list: { type: 'ordered'|'unordered'|'tasklist', contents: Inline[], items: Listing[] } }`
- ListItem structure: `{ item: Inline[] }`
- Task inline: `{ task: { checked: boolean, content: Inline[] } }` - used inside tasklist items
- Verse is `VerseInline | VerseBlock`, use `isBlockVerse()` from channel.ts to distinguish
- Blockquote is an Inline type with `{ blockquote: Inline[] }` structure
- markdown-it tokens: inline tokens use open/close pairs (e.g., `strong_open`/`strong_close`), content is in `.content`, attributes in `.attrs` array
