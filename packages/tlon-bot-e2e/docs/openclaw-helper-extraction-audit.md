# OpenClaw Helper Extraction Audit

Milestone 1 classifies the current `packages/openclaw/test/lib/*` helpers before moving anything beyond the fake model. The shared package must not depend on `@tloncorp/openclaw` internals, OpenClaw service names, or OpenClaw log formats.

| OpenClaw helper                | Classification             | Milestone 1 action               | Notes                                                                                                                                                                                                                                             |
| ------------------------------ | -------------------------- | -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `test/lib/client.ts`           | shared later after rewrite | keep in OpenClaw                 | Tlon DM client shape is reusable, but it imports OpenClaw story conversion from `src/urbit/story.js` and includes OpenClaw-specific prompt/log tracing behavior. A shared version needs neutral story conversion and driver-provided diagnostics. |
| `test/lib/config.ts`           | shared later after rewrite | keep in OpenClaw                 | Environment parsing and host/container URL normalization are reusable, but variable names and ship roles should become package-neutral runtime endpoint config.                                                                                   |
| `test/lib/docker-logs.ts`      | driver-owned               | keep in OpenClaw                 | Hard-codes the `openclaw` Docker service and parses OpenClaw `embedded run tool` / `tooltrace` log formats.                                                                                                                                       |
| `test/lib/docker-logs.test.ts` | driver-owned               | keep in OpenClaw                 | Unit coverage for the OpenClaw-owned log parser. Move only if a driver log interface is introduced later.                                                                                                                                         |
| `test/lib/fixtures.ts`         | shared later after rewrite | keep in OpenClaw                 | Fixture lifecycle is broadly useful, but profile/group names, settings buckets, approval setup, and model-engaging probe flow are OpenClaw-specific today.                                                                                        |
| `test/lib/index.ts`            | driver-owned               | keep in OpenClaw                 | Barrel file for the current OpenClaw helper surface. It should remain package-owned until helpers are split.                                                                                                                                      |
| `test/lib/post-baseline.ts`    | shared now                 | keep in OpenClaw for Milestone 1 | Pure post sequence/text helpers with only `@tloncorp/api` text extraction. Safe to move later, but not needed for fake-model smoke tests.                                                                                                         |
| `test/lib/scripted.ts`         | driver-owned               | keep in OpenClaw                 | Tied to OpenClaw's current singleton fake-model compatibility path and OpenClaw session-history behavior. Shared scenario scripting should be driver-adapted later.                                                                               |
| `test/lib/state.ts`            | shared later after rewrite | keep in OpenClaw                 | State/scry client patterns are reusable, but the implementation relies on the global `@tloncorp/api` client singleton, uses `Math.random()` for fixture IDs, and exposes OpenClaw-specific fixture operations.                                    |
